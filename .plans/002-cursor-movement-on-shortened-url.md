# 省略 URL 上のカーソル移動修正計画

## 背景

- 省略表示中の Markdown URL 上でカーソルを左右移動すると、見た目はアイコン 1 個でも内部の URL 文字数ぶんカーソル移動が発生する。
- 期待挙動は、省略後のアイコンを 1 つの表示単位として扱い、1 回のカーソル移動で通過できること。

## 原因仮説

- 現在の実装は `Decoration.replace` で URL 表示をアイコンへ置き換えている。
- ただし、カーソル移動時にその置換範囲を CodeMirror 6 の atomic range として扱っていないため、内部ドキュメント上の各文字位置を順に通過している可能性が高い。

## 修正案

1. `lib/short-link-message-dialog.js` の短縮 URL decoration 範囲を `EditorView.atomicRanges` にも提供する。
2. `Decoration.replace` と atomic range の対象範囲を同一に保つ。
3. 初期化やノート切替時にも既存 extension と整合することを確認する。
4. 必要であれば editor 監視まわりの null-safe ガードも同時に見直す。

## 追加メモ

- `atomicRanges` 追加後もカーソル移動量が変わらない報告あり。
- Inkdrop 側のキー処理で atomic range だけでは十分に反映されていない可能性がある。
- 次の対応として、短縮 URL 範囲を使って `ArrowLeft` / `ArrowRight` で隠し URL を 1 回で飛び越える keymap を追加する。

## 現状整理

- vim plugin 利用環境に合わせて、短縮 URL をまたぐ移動補正のキーは `ArrowLeft` / `ArrowRight` ではなく `h` / `l` を使う。
- このキー設定の影響範囲は、CodeMirror エディタ上でカーソルが短縮 URL の隠し範囲内にある場合に限られる。
- 短縮 URL 範囲外では、このプラグインは処理を横取りせず、既存の vim plugin / CodeMirror 側の通常挙動へ流す。

## 性能方針

- 性能面の最適化は今回の対応スコープに含めない。
- 現状実装では、選択変更時を含めて短縮 URL 範囲を再計算しているため、長文ノートでは改善余地がある。
- ただし今回追加した `h` / `l` 補正自体の負荷は小さく、今回の主目的であるカーソル移動量の修正を優先する。

## 確認項目

- 省略 URL 上を左右キーで移動したとき、アイコン 1 個ぶんの移動になること
- ノート切替後も短縮表示が維持されること
- プラグイン読み込み時に例外が発生しないこと
- `node --check lib/short-link-message-dialog.js` が成功すること
